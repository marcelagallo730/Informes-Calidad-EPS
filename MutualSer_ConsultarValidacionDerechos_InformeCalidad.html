🚨 Hallazgos Críticos y Problemas Identificados
Validez · Criticidad Alta
Campo telecom.firtPhone masivamente inutilizable
38 306 registros (80.6%) contienen el texto literal "Phone Not found" en lugar de un valor nulo. El campo aparece formalmente completo (0 nulos), pero es funcionalmente vacío en más de 4 de cada 5 filas. Este patrón de valor centinela enmascara ausencias y distorsiona métricas de completitud. Adicionalmente, el nombre del campo contiene un error tipográfico ("firtPhone" vs. "firstPhone").
Uniformidad · Criticidad Media
sisbenLevel mezcla códigos cortos y descripciones largas
El campo contiene tanto valores breves ("Nivel 1", "Nivel 2") como descripciones textuales extensas ("Afiliación de Oficio sin Encuesta Sisben ni Población Especial"), lo que impide análisis cuantitativo uniforme por nivel. Se requiere normalización a un catálogo con código + descripción separada.
💡 Recomendaciones de Gobernanza de Datos
  • 1
    Homogeneizar el tipo de dato de documentId: Establecer en el modelo de datos el tipo VARCHAR(20) para documentId, que soporte tanto documentos numéricos como los 3 valores alfanuméricos detectados. Documentar y validar el patrón '13001D000071' con el área fuente para determinar si es un código válido especial o un error de ingreso.
  • 2
    Normalizar el catálogo de sisbenLevel: Crear una tabla de referencia con código corto (ej. N1, N2, N3, NO, OF, SC) y descripción larga asociada. Reemplazar en el campo principal el texto libre por el código. Esto permitirá análisis cuantitativos por nivel y reduce el tamaño del campo en storage.
